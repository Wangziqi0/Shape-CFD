## Agent 1：AD-Rank 核心求解器

**文件**：`ad_rank.js`（新建）

**职责**：实现对流-扩散方程的图上求解器

**输入**：
```js
adRank(queryVector, docVectors, k, options)
// queryVector: Float32Array(4096)  — query 的 embedding
// docVectors:  Float32Array[](N)   — 候选文档的 embedding 数组
// k: number                        — 返回 top-k
// options: { D, uStrength, maxIter, epsilon }
```

**输出**：
```js
{
  rankings: [{ index, score, topology }],  // 排序结果
  reynolds: number,                         // 雷诺数
  iterations: number,                       // 实际迭代步数
  convergence: boolean                      // 是否收敛
}
```

**实现步骤**：
1. 计算 N×N cosine similarity 矩阵
2. KNN (k_nn=5) 建稀疏邻接矩阵 W
3. 图拉普拉斯 L = D_degree - W
4. 初始化 C₀[i] = cosine(query, doc_i)
5. 对流方向 u_ij = proj(doc_j - doc_i, query)
6. 迭代求解 C_{t+1} = C_t + Δt(-u·∇C + D·L·C)
7. 收敛判断 ||C_{t+1} - C_t|| < ε
8. 按 C_final 排序返回 top-k

**依赖**：无外部依赖，纯数学计算
  可复用 `search_engine.js` 的 `_cosineSim()` 方法

**扩展**（v2 BAA 分块模式）：在基础版验证有效后，增加分块自适应对流
  → 将 4096 维分成 B 块，每块独立建图+独立对流方向
  → 跨块融合：C_final = Σ w_b × C_b + λ × cross_block_term

## Agent 2：数据接口层

**文件**：`ad_rank_data.js`（新建）

**职责**：从现有知识库加载向量、调用 embedding API 获取 query 向量

**需要做的**：
1. 加载 USearch 索引（复用 `search_engine.js` L104-106 的 usearch.Index 初始化逻辑）
   路径：`knowledge_base/vectors/law.usearch`
2. 加载元数据（`knowledge_base/vectors/metadata.json`，JSON 数组，每项含 law/path/article/content）
3. 封装 query embedding 接口（复用 `vectorize_engine.js` 的 `embed()` 方法）
   API：`http://192.168.31.22:3000/v1/embeddings`
4. 提供 `getCandidateVectors(queryVector, preFilterK=100)` — HNSW 粗筛候选
5. 提供 `getChunkMetadata(indices)` — 根据索引返回法条文本
6. 提取候选文档的原始 4096 维向量（需从 USearch 索引中 retrieve）

**输入输出**：
```js
// 加载候选
const { vectors, metadata } = await getCandidateVectors(queryVector, 100);
// vectors: Float32Array[](100)
// metadata: [{ id, title, text, source }]

// 获取 query 向量
const qVec = await embedQuery("劳动合同解除赔偿标准");
// qVec: Float32Array(4096)
```

**依赖**：`vectorize_engine.js`, `data_engine.js`, `usearch` (npm 包), `better-sqlite3`

**⚠️ 关键问题**：USearch 的 `index.search()` 返回 `{keys, distances}`，
  但不直接返回原始向量。Agent 1 需要原始向量来建图和计算对流方向。
  解决方案：
  a) 从 metadata.json 中反查 chunk ID → 调用 embedding API 重新 embed（慢，需缓存）
  b) 单独导出一份 vectors.bin / vectors.json（一次性）
  c) 用 usearch 的 `index.get(key)` 方法（如果支持）

## Agent 3：实验框架

**文件**：`ad_rank_experiment.js`（新建）

**职责**：实现三组对比实验 A/B/C

**三组实验**：
```
组 A（基线）：纯 cosine similarity 排序
组 B（Diffusion-Aided RAG 复现）：图上纯扩散 C = C + Δt·D·L·C（无对流项）
组 C（AD-Rank）：图上对流-扩散 C = C + Δt·(-u·∇C + D·L·C)
```

**需要做的**：
1. 准备 30-50 个测试 query + 对应的正确文档（人工标注 ground truth）
2. 实现组 A：cosine 排序
3. 实现组 B：调用 Agent 1 的求解器但 uStrength=0（去掉对流项）
4. 实现组 C：调用 Agent 1 的求解器，uStrength>0
5. 计算评估指标：Top-K 命中率、MRR、NDCG@10
6. 输出对比表格（markdown）

**输出文件**：`experiment_results.md`

**依赖**：Agent 1 (`ad_rank.js`), Agent 2 (`ad_rank_data.js`)

## Agent 4：流场拓扑分析

**文件**：`ad_rank_topology.js`（新建）

**职责**：分析对流-扩散稳态场的拓扑特征

**需要做的**：
1. 输入 Agent 1 的求解结果（C_final, 图结构, 对流方向）
2. 计算每个节点的散度 div_i = Σ u_ij × W_ij × C[j]
3. 分类：
   - 汇聚点：div < -threshold（多文档支持同一结论）
   - 发散点：div > threshold（观点分歧）
   - 停滞点：|∂C/∂t| ≈ 0 且 C 值中等（语义矛盾处）
4. 计算雷诺数 Re = |对流项| / |扩散项|
5. 返回拓扑标签和 Prompt 构建建议

**输出**：
```js
{
  convergencePoints: [indices],  // 汇聚点
  divergencePoints: [indices],   // 发散点
  stagnationPoints: [indices],   // 停滞点
  reynolds: number,
  promptStrategy: 'enhance' | 'dialectic' | 'focus'
}
```

**依赖**：Agent 1 (`ad_rank.js`)

#### Agent 10 详细成果（多方向对流 MDA + 分块对流 BAA）

- **新增文件**: `ad_rank_v3.js` (adRankCore + adRankMDA + adRankBAA)、`ad_rank_v3_benchmark.js` (四组对比+盲评)
- **核心思路**: 通过降维/分块放大对流信号，解决 v2 维度诅咒 (σ=1/√4096≈0.0156)
  - **MDA**: M=8 个随机投影到 k=32 维子空间，σ=1/√32≈0.177 (11x 放大)
  - **BAA**: B=8 块 (每块 512 维)，保持 embedding 维度局部性
- **四组对比实验** (`ad_rank_v3_benchmark.js`, 10 个标准 query):

| 指标 | B (v2) | C (MDA) | D (BAA) |
|:---|:---:|:---:|:---:|
| 平均速度 | 15.7ms | 140.0ms ⚠️ | **7.7ms** ✅ |
| 平均 Pe | 0.065 | **0.351** (5.4x) | **0.226** (3.5x) |
| 对流贡献率 | ~16% | **86.0%** | **87.7%** |
| 盲评均分 | — | 2.5/5 | **3.0/5** |
| 盲评胜负 (vs cosine) | — | 3:7 输 | **5:4 赢** ✅ |

- **结论**: **BAA（分块对流）是最优方案** — 速度最快(7.7ms)、对流贡献率最高(87.7%)、唯一在 DeepSeek 盲评中胜出 cosine 的方案
- **MDA 局限**: 随机投影割裂了 embedding 的语义连续性，速度偏慢(140ms)

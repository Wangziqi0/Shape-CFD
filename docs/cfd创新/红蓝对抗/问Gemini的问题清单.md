# AD-Rank 开放问题 — 请 Gemini 分析

> 背景：我们实现了 AD-Rank（对流-扩散方程做向量检索重排序），已有实验数据。
> 以下是需要数学/理论层面分析的问题。

---

## 已有结论（实验数据，不需要再验证）

```
算法：在 KNN 图上跑对流-扩散方程重排序 30 个 HNSW 候选
向量：Qwen3-Embedding-8B, 4096维
数据：23,701 个中国法条 chunk
求解：2.2ms, 收敛 10/10

DeepSeek 盲评：AD-Rank 6:3 胜 cosine (平均 3.3/5 vs 2.8/5)

关键发现：
- 最优对流强度 uStrength=0.1（最弱档），说明对流信号弱
- Pe = 0.065（结构层面对流几乎可忽略）
- 对流仅贡献 16% 排序差异，扩散贡献 59%
- 输的 3 个 query 都是单一法域精确匹配问题
```

## 问题 1：为什么对流信号这么弱？

对流方向定义：
```
u_ij = <normalize(v_j - v_i), normalize(query)> × uStrength
```

实验中 `mean(|u_ij|)` 只有 ~0.01（Pe = mean(|u|)/D = 0.065，D=0.15）。

**猜想**：4096 维空间中，任意两个向量的差向量 `v_j - v_i` 与 query 向量几乎正交（高维空间的维度诅咒——随机向量在高维中趋向正交）。

**问 Gemini**：
1. 这个猜想是否正确？在 d=4096 维中，两个随机单位向量 dot product 的期望值和方差是多少？
2. 如果对流信号确实被维度诅咒淹没，有什么数学方法可以放大它？

---

## 问题 2：BAA 分块对流（Block-Adaptive Advection）

思路：将 4096 维分成 B 个块（比如 8×512），每块独立建图、独立算对流方向。

```
全局对流：u_ij = dot(edge_4096, query_4096) ≈ 0 (维度诅咒)
分块对流：u_ij^(b) = dot(edge_512, query_512) ← 每块信号可能更强？
```

**问 Gemini**：
1. 高维 embedding（如 Qwen3-Embedding）的不同维度子空间是否编码了不同的语义信息？还是完全纠缠？
2. 如果是纠缠的，分块后 `u_ij^(b)` 的信号是否比全局 `u_ij` 更强？数学上期望值和方差如何变化？
3. 分块后，每块的 KNN 图可能完全不同——这是优势还是劣势？
4. 融合公式 `C_final = Σ w_b × C_b` 的权重 `w_b` 应该怎么确定？等权？按块方差？按块 Pe？

## 问题 3：反对称性假设是否过强？

当前实现中 `u_ji = -u_ij`（边方向反转 → 对流方向取反）。

但实际上 `normalize(v_i - v_j) ≠ -normalize(v_j - v_i)` 在高维中可能有数值差异。
更重要的是，语义空间可能不是各向同性的——从 A 到 B 的"语义距离"不一定等于从 B 到 A。

**问 Gemini**：
1. 反对称性在数学上是否严格成立？还是只是近似？
2. 如果放弃反对称性（独立计算 u_ij 和 u_ji），对守恒性有什么影响？
3. 在 embedding 空间中，是否有理论支持"非对称语义距离"？

## 问题 4：扩散主导是好事还是坏事？

实验显示 Re ≈ 0.7，扩散远强于对流。改进主要来自扩散（59% 排序变化）。

**问 Gemini**：
1. 在信息检索语境中，"扩散主导"意味着什么？是否说明 AD-Rank 退化成了一种增强版的 Diffusion-Aided RAG？
2. 如果是这样，对流项的理论价值在哪里？仅仅是"让扩散更保守"（从 41% 重叠拉回到 52%）是否足够发论文？
3. 有没有其他方式定义"对流方向"，使其信号更强？比如用 attention 权重而非 dot product？

3. 如果只谱化扩散、保留迭代对流，混合方案的数学正确性如何？

## 问题 9 ⭐（核心灵感）：向量不是点，是形状

**这是最重要的问题。**

当前所有向量检索方法（包括 AD-Rank v2）都把每个文档视为高维空间中的**一个点**。但实际上，一段文本的语义不是一个精确的点——它是一个**不规则的高维区域/形状**。

比如"劳动合同解除赔偿标准"这个法条，它的语义至少覆盖：
- "劳动合同"的概念区域
- "解除"的动作区域
- "赔偿标准"的金额/规则区域
- 以及它们的交叉

**灵感**：

```
当前范式（v2）:
  doc = point(4096维)
  流体在点构成的 KNN 图上流动

新范式构想:
  doc = shape(高维不规则形状)
  "形状"如何定义？
    - 一个文档的多个 chunk/句子的 embedding 构成一个点云
    - 或者：一个向量 + 它的 attention 各头输出构成一个凸包
    - 或者：用 VAE/扩散模型学一个隐式密度函数 p(x|doc)
  
  流体力学在这些形状之间/之中运动：
    - 形状之间的"接触面积"定义扩散通量（不再是点-点的 cosine）
    - 形状的不规则性定义对流方向（流体绕流不规则体产生涡流）
    - query 不再是一个点，而是一束射入场域的"流"
```

**问 Gemini**：
1. 文献中有没有将向量表示为"区域/形状"而非"点"的检索方法？（如 region embedding, set embedding, density estimation 等）
2. 在 CFD 中，不规则形状之间的流体交互用什么数学工具描述？（N-S 方程 + 边界条件？浸没边界法？Level Set？）
3. 如果每个文档用一个点云（多个子向量）表示，能否定义一个有意义的"形状之间的对流-扩散"方程？
4. 计算可行性：30 个"形状"（每个形状由 5-10 个子向量定义）在 4096 维空间中做 CFD，能否在 10ms 内完成？
5. 这个方向有没有已知的理论障碍（如维度诅咒、计算不可行等）？
6. 最小可验证实验：如何用最少的代码验证"形状 CFD"是否比"点 CFD"更有效？

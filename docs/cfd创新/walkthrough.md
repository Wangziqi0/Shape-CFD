## 三、Gemini 两轮诊断要点

### 3.1 第一轮诊断（8 个问题回答）

1. **MaxSim 退化**：单向量 query 下 Chamfer = MaxSim + MeanSim，V5 丢弃了 MeanSim 惩罚项
2. **γ=0.5 数值塌陷**：C₀∈[0.1,0.2] 时 R(C)=0.004，乘 dt=0.05 后仅 0.0002，被扩散淹没
3. **θ 无差异**：反应项整体休眠导致 θ 模式的选择不可观测
4. **V4 > V2 本质**：Chamfer 是最优传输近似，赋予势能盆地拓扑结构
5. **硬截断**：5%分位在 4096 维是噪声波动，切的是信号
6. **CFL 需修正**：dt < min(CFL_adv, CFL_diff, 2/(γ·max|R'|))
7. **适用场景**：长 query + 长文档（TREC-Robust04, COLIEE）
8. **Ensemble 隐患**："最大展开率"偏好发散参数

### 3.2 第二轮诊断（V5.1 失败分析）

> [!IMPORTANT]
> **核心洞察：Min-Max 把 [0.1,0.2] 拉到 [0,1] = 空间拉伸 10×，γ=5 等效于原空间 γ=500**

**V5.2 处方（已实验验证）：**

1. **不归一化** — 保持物理自然尺度 ✅ 有效
2. **域适应 γ_eff**：
   ```
   γ_eff = γ_base / (Cmax - Cmin + ε)²
   R(C) = γ_eff · (C - Cmin)(Cmax - C)(C - θ_Otsu)
   ```
   - γ_base = 0.5~2.0 → 结果无差异，说明反应项贡献极微
   - CFL 恒稳：验证通过
3. **Zelnik-Manor 局部 σ**：✅ 已实现，对 top-30 效果中性
4. **拓扑扩容**：❌ top-100~200 + cosine 图 = -13%，cosine 图质量远不如 Chamfer 图

## 三-C、PQ-Chamfer 子空间距离 (2026-03-28)

### 核心思想

将 4096d cosine 切为 64 个 64d 子空间，独立计算 cosine 距离后取均匀平均。缓解高维浓度效应（Concentration of Measure）提升区分度。

灵感来源：AMX tile 架构 / Zen5 Chiplet CCD 独立计算。

### 消融实验 (NFCorpus, 323 queries)

**[A] 切分数量** — 更多更小的子空间更好:
- 16×256: 0.2650 | 32×128: 0.2684 | **64×64: 0.2729 🏆**

**[B] 加权方式** — 均匀加权胜出:
- 均匀: 0.2684 | query 能量加权: 0.2586 (-3.7%)

**[C] 维度顺序** — 无影响:
- 顺序: 0.2684 | 打乱: 0.2681 (-0.1%)

## 三-G、V11 Token 级点云实验 (2026-03-31)

### 核心思想

从 Qwen3 的句子级嵌入（每文档几个句子向量）推进到 Token 级嵌入（每文档几百个 token hidden states），构成密集点云。Token-level PQ-Chamfer 本质上是一种无参数 cross-attention。

### 实现方案

1. **Token 提取**：llama.cpp --pooling none，提取 Qwen3-Embedding-8B 的 per-token hidden states（4096 维）
2. **存储**：token_clouds.sqlite，14GB，880791 tokens
3. **距离计算**：Rust PQ-Chamfer 全库扫描（64x64 子空间切分）
4. **评测**：NFCorpus 323 queries 全量

### 实验结果

| Method | NFCorpus NDCG@10 | vs cosine |
|:-------|:-------:|:---------:|
| cosine（基线） | 0.2195 | — |
| Rust shapeCfdPipeline（句子级最优） | 0.2852 | +29.9% |
| **V11 Token Chamfer（直排）** | **0.3214** | **+46.4%** |
| V11 Token Chamfer + PDE | 0.3180 | +44.9% |

### 分析

1. **范式级突破**：从 0.2852 跳到 0.3214，单步提升 +12.7%，累计 +46.4% vs cosine
2. **纯 Chamfer 直排 > PDE 管线**：0.3214 > 0.3180，token 级的密集点云已经提供了足够的语义分辨率，PDE 的图扩散反而引入噪声
3. **Token-level Chamfer = 无参数 cross-attention**：每个 query token 在文档 token 中找最近邻，等效于硬注意力聚合
4. **延迟**：436ms/query，主要瓶颈在全库 token 扫描（880K tokens），待优化到 <100ms
5. **数据量**：14GB SQLite，比句子级（~2.8GB）增长约 5x，但换来 +46.4% 的质量飞跃

## 三-H、V11 Fusion 消融实验 (2026-03-31)

### 核心思想

在 token_2stage 无损加速（305ms→23ms）完成后，探索 PDE 的回归路径。发现句子级 PDE 可以作为正交信号源与 token Chamfer 融合，而非替代。

### 融合公式

score = lambda * token_2stage_score + (1-lambda) * shape_cfd_v10_score

### 完整消融表（NFCorpus 323 queries）

| 方法 | NDCG@10 | vs cosine | 延迟 |
|:-----|:-------:|:---------:|:----:|
| cosine 基线 | 0.2195 | — | 20ms |
| shape_cfd_v10（句子级PDE） | 0.2852 | +29.9% | 26ms |
| token_chamfer（全库扫描） | 0.3214 | +46.4% | 305ms |
| token_v11_pipe（token粗筛+句子PDE） | 0.3180 | +44.9% | 316ms |
| token_2stage_100（质心粗筛+精排） | 0.3220 | +46.7% | 23ms |
| sampled_graph PDE（token采样建图） | 0.3218 | +46.6% | 847ms |
| fusion_03 (lambda=0.3) | 0.3100 | +41.2% | ~26ms |
| fusion_05 (lambda=0.5) | 0.3231 | +47.2% | ~26ms |
| **fusion_07 (lambda=0.7)** | **0.3232** | **+47.2%** | **~26ms** |
| weak_pde (D=0.01) | 0.3211 | +46.3% | 314ms |

### 分析

1. **PDE 以融合方式回归**：fusion_07(0.3232) > token直排(0.3214)，+0.18 个百分点
2. **PDE 价值定位：正交信号源做集成，而非同一视角平滑**
   - token Chamfer 看"局部最近邻匹配质量"
   - 句子级 PDE 看"文档图上的全局语义传播结构"
3. **图-浓度不匹配已确认**：sampled_graph(0.3218) 用 token 数据重建图后 PDE 仍无额外增益 → PDE 价值在于不同视角
4. **extreme弱PDE验证数学预测**：D=0.01 时 PDE 几乎不扩散，0.3211 ≈ 0.3214，保序性确认
5. **最优融合延迟仅26ms**：token_2stage(23ms) 和 shape_cfd(26ms) 可并行

> [!IMPORTANT]
> **全场最优**：fusion_07 = **0.3232 (+47.2% vs cosine)**
> **融合公式**：score = 0.7 * token_2stage + 0.3 * shape_cfd_v10
> **延迟**：~26ms（两路并行）
> **PDE 定位**：正交信号源做集成，而非同一视角平滑
